# 釐清問題

停用使用者主檔時，審計需記錄「操作者」來源（人員帳號、API client、服務帳號）與「理由」。欄位與出處如何統一？

# 定位

Feature：使用者主檔管理／規則：停用與審計

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 新增 `audit` 統一結構：actor_type、actor_id、reason、at |
| B | 僅在 `audit_log` JSON 追加自由格式紀錄 |
| C | 事件（ChangeFeed）包含審計摘要，儲存層僅保存必要欄位 |
| Short | 其他（<=5 字） |

# 影響範圍

- 稽核一致性與查詢
- 安全與合規（可追溯性）
- 事件與資料庫欄位對齊

# 優先級

High
