# 釐清問題

行為事件 `payload` 的 schema 由誰維護？是否需要跨專案共用的版本策略（`event_type` × `version`）與驗證來源？

# 定位

Feature：統一行為事件紀錄／規則：schema 驗證與版本化

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 中央維護（user-core）提供 schema registry 與驗證 |
| B | 各子專案自行維護，各自驗證（user-core 只保存） |
| C | 混合：核心事件由中央維護，自定義事件由專案維護 |
| Short | 其他（<=5 字） |

# 影響範圍

- 寫入 API 的驗證流程與錯誤訊息
- 相容性與回溯查詢
- 文檔與測試的覆蓋策略

# 優先級

High

---
# 解決紀錄

- **回答**：C - 混合治理：核心事件由 user-core 維護與驗證；自定義事件由子專案自管
- **更新的規格檔**：spec/spec.md（於「使用者故事 2」下新增「事件 schema 治理（混合模式）」）
- **變更內容**：定義核心事件的中央 registry 與寫入驗證、最低欄位規範，以及自定義事件之追溯欄位（schema_url/version）
